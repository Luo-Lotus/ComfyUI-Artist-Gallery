# ArtistSelector 高级配置功能 - 实现总结

## 已完成的功能

### ✅ 核心功能
1. **格式化系统**
   - 支持 `{content}` 变量替换画师名称
   - 支持 `{random(min,max,step)}` 生成随机数
   - 实时预览格式化结果
   - 格式验证功能

2. **多画师随机规则**
   - 可配置从已选画师中随机选择 N 个
   - 支持禁用（使用所有已选画师）

3. **循环模式**
   - 循环使用已选画师，每次输出一个
   - 后端持久化存储循环状态
   - 支持重置循环位置

4. **分类配置覆盖**
   - 每个分类可独立配置
   - 支持覆盖全局配置或恢复使用全局配置
   - 配置指示器显示哪些分类有独立配置

### ✅ UI组件
- **GlobalConfigDialog** - 全局配置对话框
- **CategoryConfigDialog** - 分类配置对话框
- **配置按钮** - 已选择区域的全局配置按钮
- **分类配置按钮** - 每个分类标签上的独立配置按钮

### ✅ 数据持久化
- localStorage 保存配置（前端）
- 后端内存存储循环状态（后端API）

## 使用说明

### 1. 全局配置
1. 在 ArtistSelector 节点的已选择区域，点击 **⚙️ 全局配置** 按钮
2. 配置选项：
   - **单画师格式**：设置画师名称的输出格式
     - 示例：`({content}:{random(1,1.5,0.1)})` → `(artist:1.3)`
   - **多画师随机规则**：启用后从已选画师中随机选择 N 个
   - **循环模式**：启用后每次输出一个画师，依次循环
3. 点击保存

### 2. 分类配置
1. 在已选择的分类标签上，点击 **⚙️** 按钮
2. 勾选 **使用独立配置（覆盖全局配置）**
3. 配置该分类的格式和随机规则
4. 点击保存
5. 配置后，分类标签上会显示 **●** 指示器

### 3. 格式字符串语法

#### 变量
- `{content}` - 画师名称

#### 随机函数
- `{random(min,max,step)}` - 生成随机数
  - min: 最小值
  - max: 最大值
  - step: 步长（可选，默认 0.1）

#### 示例
```
{content}
→ artist_name

({content}:1.2)
→ (artist_name:1.2)

({content}:{random(1,1.5,0.1)})
→ (artist_name:1.3)

{content}_{random(1,100,1)}
→ artist_name_57
```

### 边界情况
1. 无已选画师时的配置
2. 格式字符串为空
3. 随机数范围错误（min > max）
4. 循环状态丢失（后端重启）
5. 分类配置被删除后回退到全局配置

## 注意事项

### 1. 配置存储
- **前端配置**（格式、随机规则）存储在 localStorage
- **循环状态**存储在后端内存中（服务器重启后丢失）

### 2. 循环模式限制
- 循环状态按节点ID存储
- 服务器重启后循环位置会重置
- 建议在稳定环境中使用

### 3. 性能考虑
- 格式解析结果会被缓存
- 随机数在每次输出时生成
- 大量画师时循环模式性能最优

### 4. 浏览器兼容性
- 使用了 ES6+ 语法
- 需要现代浏览器支持
- localStorage 需要启用
